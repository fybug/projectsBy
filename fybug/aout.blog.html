专门用与 giteePage , githubPage 等静态托管服务中的自动化生成伪动态博客数据的 PDWebpack 插件。

markdown 内容头部

++++++
title: 标题
date: 2020-06-16 8:00:00
tag: 标签, 用, 分割
des: 描述，**支持 markdown 语法**
link: 博客点击后的链接
label: 博客的小卡片的字符串
icon: 图标
++++++

头部数据作为重要的展示数据，在生成页面列表时获取，仅会提供数据，自行填充数据到页面中

头部数据用于生成上面 demo 展示的部分

凡是单纯字符串的都可以自己做格式和解析，实现个性化。

  • title：标题，单纯的字符串数据, 对应上面 demo 中的 aout.blog 字段数据
  • date：文章日期，在打包时会转换为 Date 对象再转回字符串，打包时会根据这个进行排序, 对应上面 demo 中的 日期 字段数据
  • des：描述，支持 markdown 的字符串数据，会打包为解析后的 html 字符串数据, 对应上面 demo 中的 最后一行描述 字段数据
  • link：博客点击后的链接，也是单纯的字符串数据，需要自己生成链接对象
  • label：也是单纯的字符串数据
  • icon：同样是单纯的字符串数据，需要自己生成图片节点, 对应上面 demo 中的 图标
  • tag：标签数据，打包时会根据此数据归纳标签页数据和单独标签的内容列表，使用 , 号分割。打包后会生成数组，而且不是单纯的字符串

编译后的数据呈现多个js 文件，主要数据按照页码存放，建议通过与服务器同样的交互方式获取

如何使用生成的数据，请看wiki。

使用

下载库的压缩文件，通过 发行版 下载，带 _bin 后缀的是带有依赖库的。

安装方式参考 wiki 文档

需要配合使用 PDWebpack 手脚架才可发挥作用

在 PDWebpack 的环境中追加安装：

npm install fs globby marked path --save-dev
